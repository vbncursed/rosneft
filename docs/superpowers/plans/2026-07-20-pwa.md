# PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Сделать фронтенд устанавливаемым приложением на macOS, Windows, Linux, Android и iPadOS — манифест, иконки, минимальный service worker, viewport.

**Architecture:** Ни одной новой зависимости. Геометрия знака живёт в одном модуле как строка SVG; два route-файла в `src/app/` рендерят из неё PNG через `ImageResponse` из `next/og`. Service worker перехватывает только навигацию и при сетевой ошибке отдаёт закэшированную `/offline` — `/api/*`, SSE и потоковая отдача GLB его не касаются.

**Tech Stack:** Next.js 16.2.10 (App Router), React 19, TypeScript strict, `next/og`, node:test.

## Global Constraints

- Спека: `docs/superpowers/specs/2026-07-20-pwa-design.md`. Ветка: `feat/pwa`.
- Все команды выполняются из `frontend/`.
- **Жёсткий лимит 200 строк на файл** (ESLint `max-lines`, skipBlankLines, skipComments).
- Clean Architecture: файл живёт в одном из слоёв `domain/` `application/` `infrastructure/` `presentation/` внутри своего контекста. Исключение — `src/app/**` (роутинг Next).
- **Слово «Роснефть»/«Rosneft» запрещено в отображаемом тексте.** Бренд — «Andrey».
- Цвета знака заданы жёстко: рельеф `#2FD79A`, отметка `#FF7A45`, фон плитки `#0E1211`. Браузер рендерит иконку вне CSS-контекста страницы — `currentColor` и медиазапросы там не работают.
- Цвета темы: light `#ffffff`, dark `#0a0a0a` — совпадают с `--background` в `src/app/globals.css`.
- Никакой подготовки к Tauri. Кода она не требует.

## Отклонение от спеки (согласовано при планировании)

Спека называла `src/app/icon.svg` статическим файлом. В плане вместо него —
`src/app/icon.tsx`, рендерящий PNG из общего модуля `app-mark.ts`. Причина:
статический `.svg` заставил бы держать геометрию знака в двух местах
(отдельно для вкладки, отдельно для `apple-icon`). Через общий модуль
источник один. Роль вектора во вкладке продолжает исполнять существующий
`src/app/favicon.ico`, который остаётся нетронутым.

## О тестах

Проверено на этапе планирования: `next/server` не резолвится под
`node --test` (`ERR_MODULE_NOT_FOUND`), поэтому `middleware.ts` юнит-тестом не
покрывается. Вынести matcher в отдельный модуль тоже нельзя — Next
анализирует `config.matcher` статически и принимает только литерал.

Тестируется то, что реально тестируется: `manifest()` — чистая функция,
её единственный импорт из Next это `type`, который стирается при
компиляции. Остальное проверяется одним ручным прогоном в браузере
(Задача 5) — вся логика в браузерных API, юнит-тест там не имеет предмета.

- [ ] **Step 1: Создать модуль знака**

Файл `frontend/src/shared/presentation/app-mark.ts`:

```ts
// Знак приложения: каркас многогранника, пять вершин цветом рельефа и верхняя
// вершина цветом отметки — так же, как выделенное размещение в редакторе.
//
// Цвета жёсткие: браузер рендерит иконку вне CSS-контекста страницы, поэтому
// currentColor и prefers-color-scheme здесь не работают. Светлые версии
// читаются и на светлой, и на тёмной панели вкладок.
const RELIEF = "#2FD79A";
const MARKER = "#FF7A45";

export const MARK_SVG = `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 64 64">
  <g fill="none" stroke="${RELIEF}" stroke-width="4" stroke-linejoin="round" stroke-linecap="round">
    <path d="M32 8 11 20v24l21 12 21-12V20Z"/>
    <path d="M32 8v24M32 32 11 44M32 32l21 12" stroke-width="3" opacity=".5"/>
  </g>
  <g fill="${RELIEF}">
    <circle cx="11" cy="20" r="4.5"/><circle cx="53" cy="20" r="4.5"/>
    <circle cx="11" cy="44" r="4.5"/><circle cx="53" cy="44" r="4.5"/>
    <circle cx="32" cy="56" r="4.5"/>
  </g>
  <circle cx="32" cy="8" r="6" fill="${MARKER}"/>
</svg>`;

/** Фон плитки на iPadOS: альфа там не поддерживается, нужен непрозрачный цвет. */
export const MARK_BACKDROP = "#0E1211";

export function markDataUri(): string {
  return `data:image/svg+xml;base64,${Buffer.from(MARK_SVG).toString("base64")}`;
}
```

- [ ] **Step 1: Написать service worker**

Файл `frontend/public/sw.js`. Обратите внимание на сужение: перехватывается
только навигация, поэтому BFF-роуты `/api/*`, SSE-поток job-событий и отдача
GLB работают ровно как без service worker — он не может сломать ни
авторизацию, ни докачку ассетов.

```js
// Минимальный service worker: существует ради двух вещей — Chrome и Edge не
// показывают кнопку «Установить» без fetch-хандлера, и без сети приятнее
// увидеть свой экран, чем страницу ошибки браузера.
//
// ponytail: кэшируется только /offline. Модели весят сотни мегабайт и
// требуют политики вытеснения — добавить, если появится жалоба на скорость
// повторных заходов.
const CACHE = "andrey-shell-v1";
const OFFLINE_URL = "/offline";

self.addEventListener("install", (event) => {
  event.waitUntil(
    caches.open(CACHE).then((c) => c.add(OFFLINE_URL)).then(() => self.skipWaiting()),
  );
});

self.addEventListener("activate", (event) => {
  event.waitUntil(
    caches
      .keys()
      .then((keys) => Promise.all(keys.filter((k) => k !== CACHE).map((k) => caches.delete(k))))
      .then(() => self.clients.claim()),
  );
});

self.addEventListener("fetch", (event) => {
  // Только переходы по страницам. Всё остальное идёт мимо кэша в сеть.
  if (event.request.mode !== "navigate") return;
  event.respondWith(
    fetch(event.request).catch(async () => {
      const cached = await caches.match(OFFLINE_URL);
      // Кэш мог не наполниться — тогда отдаём штатную ошибку браузера.
      return cached ?? Response.error();
    }),
  );
});
```

- [ ] **Step 2: Написать регистрацию**

Файл `frontend/src/app/sw-register.tsx`:

```tsx
"use client";

import { useEffect } from "react";

/**
 * Регистрирует service worker после гидратации. Ничего не рендерит.
 * Регистрация молча не происходит без HTTPS, в приватном режиме и в старых
 * браузерах — приложение тогда работает как обычный сайт, установка просто
 * недоступна.
 */
export default function SwRegister() {
  useEffect(() => {
    if (!("serviceWorker" in navigator)) return;
    navigator.serviceWorker.register("/sw.js").catch(() => {
      // Молча: работоспособность приложения от этого не зависит.
    });
  }, []);
  return null;
}
```
