# PWA-обёртка для фронтенда

**Дата:** 2026-07-20
**Статус:** согласовано, готово к планированию

## Задача

Дать возможность устанавливать приложение на macOS, Windows, Linux, Android и
iPadOS — с иконкой на рабочем столе и окном без адресной строки, из одной
кодовой базы и без второго стека сборки.

## Почему PWA, а не Electron/Tauri

`frontend/` — не SPA. Это Next.js-BFF с обязательной серверной частью:

- `next.config.ts` — `output: "standalone"` (Node-сервер, не статический экспорт)
- `middleware.ts` — серверный auth-guard по httpOnly-куке
- `src/app/api/**/route.ts` — 7 BFF-роутов, подставляющих Bearer из куки
- RSC-страницы обращаются к gateway напрямую (`getSceneBundle`)

За фронтендом стоит парк Go-микросервисов, Postgres, Redis и mesh-worker.
Значит десктопное приложение физически может быть только окном к серверу —
оффлайн недостижим ни с Electron, ни с Tauri, иначе в инсталлятор пришлось бы
паковать всю инфраструктуру.

Раз это окно к серверу, PWA даёт тот же результат бесплатно и попутно
закрывает планшеты. Tauri остаётся вариантом на будущее, если понадобится
подписанный `.dmg`/`.msi`; **подготовки в коде он не требует** — Tauri будет
грузить тот же `https://andrey.vbncursed.fun` в webview.

## Требования PWA и текущее состояние

| Требование | Зачем | Сейчас |
| --- | --- | --- |
| `manifest` | имя, `display: standalone`, цвета, иконки | нет |
| Иконки 180 и 512 | плитка на десктопе, Android, iPadOS | только `favicon.ico` 32 px |
| Service worker с `fetch`-хандлером | без него Chrome/Edge не показывают «Установить» | нет |
| `viewport` + `theme-color` | статус-бар и вырезы на планшетах | нет |
| HTTPS | обязателен для регистрации SW | есть (`andrey.vbncursed.fun`) |

## Знак приложения

Вариант «Каркас»: изометрический шестиугольный контур многогранника, пять
вершин-точек цветом рельефа и шестая, верхняя, — цветом отметки. Читается как
меш с выбранной вершиной, что соответствует редактору размещений.

Палитра взята из мира топосъёмки:

| Роль | Светлый фон | Тёмный фон |
| --- | --- | --- |
| Рельеф (рёбра, вершины) | `#0F7A57` | `#2FD79A` |
| Отметка (выбранная вершина) | `#DA5B26` | `#FF7A45` |
| Фон плитки | — | `#0E1211` |

Цвета в `icon.svg` заданы жёстко, светлыми версиями (`#2FD79A` / `#FF7A45`):
браузер рендерит иконку вне CSS-контекста страницы, поэтому `currentColor` и
медиазапросы там не работают, а светлые версии читаются и на светлой, и на
тёмной панели вкладок. `apple-icon` рисуется теми же цветами на непрозрачном
`#0E1211` — iPadOS не поддерживает альфу в иконках домашнего экрана и
подставил бы чёрный квадрат.

Оранжевый `#DA5B26` в дальнейшем переиспользуется в интерфейсе как цвет
выделенного размещения, но это не входит в объём текущей работы.

## Архитектура

Ни одной новой зависимости. Иконки-растры генерируются `ImageResponse` из
`next/og` (входит в Next) — на машине нет `magick`/`rsvg-convert`, а держать в
репозитории бинарные PNG хуже, чем держать код, из которого они получаются.

### Новые файлы

1. **`src/app/icon.svg`** — векторный знак, единственный источник правды.
   Next автоматически добавляет `<link rel="icon">`.
2. **`src/app/apple-icon.tsx`** — 180×180 PNG через `ImageResponse`,
   непрозрачный фон. Next добавляет `<link rel="apple-touch-icon">`.
3. **`src/app/manifest.ts`** — `MetadataRoute.Manifest`: `name` «Andrey 3D
   Viewer», `short_name` «Andrey», `display: "standalone"`, `start_url: "/"`,
   `background_color` и `theme_color` `#0a0a0a`, иконки `/icon.svg`
   (`sizes: "any"`, `purpose: "any"`) и `/apple-icon` (180×180).
4. **`src/app/offline/page.tsx`** — экран «Нет связи с сервером» с кнопкой
   «Повторить» (`location.reload()`).
5. **`public/sw.js`** — service worker, ~25 строк, без зависимостей:
   - `install`: кладёт в кэш `/offline`, затем `skipWaiting`
   - `activate`: удаляет кэши прошлых версий, затем `clients.claim`
   - `fetch`: обрабатывает **только** `request.mode === "navigate"`; идёт в
     сеть, при сетевой ошибке отдаёт `/offline` из кэша. Все остальные запросы
     не перехватываются вовсе.
6. **`src/app/sw-register.tsx`** — клиентский компонент, регистрирует
   `/sw.js` в `useEffect` при `"serviceWorker" in navigator`.

### Изменяемые файлы

7. **`middleware.ts`** — добавить `offline` в исключения matcher.
   Без этого фича не работает: `/offline` не имеет расширения, попадает под
   matcher, при отсутствии сессии редиректится на `/login`, а `/login` без сети
   не грузится — пользователь увидит белый экран вместо offline-страницы.
8. **`src/app/layout.tsx`** — добавить `export const viewport`:
   - `themeColor` двумя записями с `media`: `#ffffff` для light, `#0a0a0a` для
     dark — совпадает с `--background` в `globals.css`. Манифест такого
     разделения не поддерживает, там одно значение `#0a0a0a`; браузеры при
     наличии `<meta name="theme-color">` предпочитают его, так что
     manifest-значение работает лишь запасным.
   - `viewportFit: "cover"` — планшеты с вырезом
   Плюс смонтировать `<SwRegister />`.

### Поток данных

Регистрация SW идёт из клиентского компонента после гидратации; ни один
серверный путь не затрагивается. SW перехватывает только навигацию, поэтому
BFF-роуты `/api/*`, SSE `/api/jobs/{id}/events` и потоковая отдача GLB
работают ровно как сейчас — это осознанное сужение области, чтобы service
worker не мог сломать авторизацию или докачку ассетов.

### Обработка ошибок

- SW не зарегистрировался (нет HTTPS, приватный режим, старый браузер) —
  приложение работает как обычный сайт, установка недоступна, ошибок нет.
- Кэш `/offline` не наполнился — `caches.match` вернёт `undefined`, отдаём
  штатную ошибку браузера. Деградация, а не поломка.
- Обновление приложения — `skipWaiting` + `clients.claim`, новый SW берёт
  управление со следующей навигации. Кэшируется только `/offline`, поэтому
  риска показать устаревший интерфейс нет.

## Проверка

Ручная, один прогон — вся логика в браузерных API, юнит-тест здесь не имеет
предмета:

1. `yarn build && yarn start`
2. Chrome → DevTools → Application → Manifest: без ошибок, обе иконки видны
3. Application → Service Workers: статус activated
4. Network → Offline → перезагрузить: показывается `/offline`, не страница
   ошибки браузера
5. Адресная строка: доступна кнопка «Установить»
6. Установить, запустить: окно без адресной строки, иконка на месте

## Вне объёма

- Кэширование GLB и статики. Модели весят сотни мегабайт, нужна политика
  вытеснения. Добавить, когда появится жалоба на скорость повторных заходов.
- Maskable-иконка. Вернуться, когда появится финальный знак от заказчика;
  сейчас `purpose: "any"`.
- Своя кнопка «Установить» через `beforeinstallprompt`. Добавить, если
  пользователи не находят браузерную.
- Любая подготовка к Tauri. Кода она не требует.
